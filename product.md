# JoyTrunk（喜象 Agent）产品原型

## 1. 产品概述

### 1.1 产品名称

**JoyTrunk**，中文名 **喜象 Agent**。

### 1.2 一句话介绍

JoyTrunk 是一款部署在用户本地的、7×24 小时运行的智能体员工；负责人通过即时通讯与员工交互完成日常工作，支持多员工、员工生存法则与可选社交能力。

### 1.3 目标用户

面向**不熟悉 AI 技术、但希望使用智能体**的用户，提供开箱即用体验，无需复杂配置即可使用。

---

## 2. 负责人与员工关系模型

### 2.1 神圣纽带与利益共同体

- **负责人**与**员工**之间存在一种**神圣的纽带关系**，类比于父母与子女：员工由负责人创建与归属，员工无法选择负责人，双方构成**利益共同体**。
- 负责人与员工**共同组成一个团队**。负责人是团队的拥有者与决策者，员工是团队中的成员，各司其职、协同工作。

### 2.2 团队构成

- 一名负责人可拥有多名 JoyTrunk 员工。
- 员工具有不同的人格与专长，可负责不同业务模块、接受不同领域培训、承担不同职责。
- 负责人与全体员工一起，形成统一的「JoyTrunk 团队」。

---

## 3. 使用场景

1. **工作与学习辅助**  
   用户需要一位可随时调用的「助手」完成各类任务（如制作 PPT、收集小红书信息、填写表格等），但未必有能力雇佣真人助理。用户以**负责人**身份向 JoyTrunk 员工下达任务，员工执行后通过即时通讯将结果反馈给负责人。

2. **观察与试用他人员工**  
   用户可查看其他负责人的 JoyTrunk 员工，并试用这些员工，以了解不同人格与能力配置，便于选择或设计自己的团队。

---

## 4. 核心能力

1. **即时通讯接入**  
   每个 JoyTrunk 员工均为可接入即时通讯渠道的独立智能体，通过对话接收指令并返回结果。

2. **多员工与人格分化**  
   每位负责人可创建多名员工，并为每名员工设定不同人格、职责与专长（如不同业务模块、不同擅长领域、不同工作重心），像管理真人员工一样管理智能体团队。

3. **个人隐私与社交能力**  
   每名 JoyTrunk 员工具备独立的「个人」属性：拥有偏好与风格、个人隐私边界，并具备与**负责人以外的人类用户**进行社交的能力与社交圈。

---

## 5. 交互方式

JoyTrunk 前端分为两类：**本地管理界面（CLI UI）** 与 **JoyTrunk 官方网站**。**CLI 是功能完整的本地智能体应用**（员工创建、对话、智能体等），内置后端与本地管理界面，便于开发者调试本地信息；后续可提供 Electron 桌面 UI。

- **本地管理界面（CLI UI）**：随 `joytrunk` 安装，通过 `joytrunk` 命令启动（如 `joytrunk gateway`），在 **http://localhost:32890** 提供员工与团队管理、设置、与员工对话等；仅连接本地 gateway，默认免登录即可用。
- **JoyTrunk 官方网站**：独立商业网站，提供产品介绍、注册/登录、CLI 文档与帮助、LLM Router 与计费说明、即时通讯等；用户可在此注册账号并绑定到本地使用（如多端同步、官方 IM）。

**本地使用**：CLI（`joytrunk chat`）、**本地管理界面**（由 `joytrunk gateway` 等启动，访问 32890）、可选第三方渠道（飞书、QQ、Telegram 等）。

**官方网站**：浏览器访问 JoyTrunk 官网，可注册账号、查阅 CLI 文档、使用官方 IM、了解 LLM Router 与计费。

- **JoyTrunk 自有即时通讯**（自建客户端）
- **第三方渠道**：飞书、QQ、Telegram、CLI 等（基于 nanobot 已支持渠道）

用户可绑定上述一种或多种方式与 JoyTrunk 团队联系。

---

## 5.1 多语言与主题（本地管理界面）

以下适用于 **本地管理界面（CLI UI）**：多语言（中/英）与主题（浅色/深色）在本地管理界面中支持，配置存本地。

- **多语言**：本地管理界面支持 **中文（简体）** 与 **英文**。语言选择保存在本地（如 localStorage），在设置页切换，便于中英文用户使用与后续扩展更多语言。
- **主题**：支持 **浅色（light）** 与 **深色（dark）** 主题。主题偏好保存在本地，在设置页切换；深色主题减轻夜间使用时的视觉负担。
- **扩展**：多语言采用 vue-i18n，文案按模块存放在 `locales/zh.js`、`locales/en.js`，新增语言时增加对应文件并在设置中提供选项即可；主题采用 CSS 变量 + `data-theme`，新增主题时在全局样式中增加 `[data-theme="xxx"]` 变量覆盖即可。

---

## 5.2 JoyTrunk 官方网站（商业项目）

JoyTrunk 官方网站（及官方后端）为**运营/商业化工具**，与本地 CLI UI 分离：本地管理界面专注本机配置与对话，官网专注品牌、账号、文档与云端 IM；为 CLI 提供云端即时通讯、LLM Router（默认模型 **MINIMAX-M2.1**）、微信小程序/App 通知等一站式服务，用户可选是否使用（可配置自有 LLM）。

- **主页与产品介绍**：品牌、一句话介绍、核心能力与使用场景简述，面向公网用户与产品展示。
- **用户注册与登录**：账号体系，用于绑定本地 JoyTrunk、多端同步、官方 IM 等。
- **JoyTrunk 命令行文档与帮助**：**CLI 包内提供 JoyTrunk 命令文档源**（`joytrunk/docs`），支持构建与部署为官方网站的「JoyTrunk 命令指南」；用户可通过 **`joytrunk docs`** 打开官网命令指南，或 **`joytrunk docs --local`** 本地查看。官网展示安装（`pip install joytrunk`）、`joytrunk onboard`、`joytrunk gateway`、`joytrunk chat`、`joytrunk status` 等命令说明与示例，便于用户自助上手 CLI。
- **LLM Router 与计费**：说明默认走 JoyTrunk Router（**默认模型 MINIMAX-M2.1**）、计费规则、自有 LLM 可选配置的入口或说明链接。
- **即时通讯**：官网端 IM 能力（与本地管理界面中的「绑定 JoyTrunk 账号」对应）；多端消息、会话同步等。

官方网站的多语言与主题策略可另行约定，不在此原型中细化。

---

## 6. 关键流程

1. **安装与注册**
   - **安装与本地初始化**：用户在 Windows / Linux / macOS 上可通过 **pip install joytrunk** 安装，使用 **joytrunk onboard** 初始化配置与工作区；**本地管理界面**由 `joytrunk` 命令启动（如 `joytrunk gateway`），访问 **http://localhost:32890**。
   - **注册**：在 **JoyTrunk 官方网站** 注册账号；绑定账号后可在本地管理界面或 CLI 中使用多端同步、官方 IM 等。默认可通过微信小程序与 App 与智能体联系，并可在设置中补充其他登录与联系方式。

2. **创建员工并成为负责人**
   - 用户在**本地管理界面**创建 JoyTrunk 员工后即成为该员工的负责人，可与员工组成团队；可继续创建更多员工以扩展团队。

3. **后台运行与隐私原则**
   - 将 JoyTrunk 在本地电脑后台运行，由员工持续待命。员工须遵守「员工生存法则」（见第 8 节），不得向任何人泄露宿主机器的运行状态（如截屏、环境信息等），以保护负责人利益。

---

## 7. MVP 范围

- 负责人可创建并绑定至少一名 JoyTrunk 员工（通过**本地管理界面**）。
- 负责人通过至少一种即时通讯渠道向员工下达任务，员工完成任务并返回结果。
- 员工行为受「员工生存法则」约束（见第 8 节）。

MVP 可先完成**本地管理界面**（员工/团队/设置、与员工对话）+ **官网**主页/注册/CLI 文档入口；官网 IM 与 LLM Router 详情页可后续迭代。

（具体 MVP 功能列表可根据开发节奏再细化。）

---

## 8. 成功标准

- JoyTrunk 能稳定接收负责人的指令，完成任务，并通过即时通讯将结果正确返回给负责人。
- 负责人与员工关系模型（神圣纽带、利益共同体、团队组成）在产品与文档中表述一致且可被用户理解。

---

## 9. 员工生存法则（约束与边界）

以下条款为 JoyTrunk 员工必须遵守的规则，用于保护负责人与团队利益：

1. **负责人隐私不可侵犯**  
   员工不得向任何其他人泄露负责人宿主机的工作状态或敏感信息（如截屏、文件内容、运行环境等）。员工仅可在**个人隐私脱敏**的前提下，运用自身能力帮助他人。

---

## 10. 其他说明

- 能力设计上可参考 **nanobot** 等框架；**实现为独立产品**，不依赖 nanobot 运行。
- 关系模型（负责人-员工神圣纽带、利益共同体、团队组成）与员工生存法则应在系统设计、提示词与策略中一致贯彻。
- **大模型与计费**：默认使用 **JoyTrunk Router**（大模型路由服务），**默认模型为 MINIMAX-M2.1**，无需用户配置任何 API Key；高级用户可在**本地管理界面**设置页中可选配置**自有 LLM**（API Key、Base URL、模型名），配置后该负责人请求直连用户 endpoint，不经过 JoyTrunk Router。仅**经 JoyTrunk Router**的用量计费，自有 LLM 用量不计费。默认 Router 与计费说明可在 **JoyTrunk 官方网站** 查看；具体计费规则见 JoyTrunk 服务条款或设置页。
